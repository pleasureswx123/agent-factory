# AI Director Agent Studio 产品需求方案

日期：2026-06-23  
基于文档：`docs/AI-Director-Agent-Product-Thinking.md`  
产品定位：导演驱动的视频 Agent 工作台

## 1. 产品概述

AI Director Agent Studio 是一个面向 AI 视频生产的 Agent 工作台。它不是普通文生视频工具，而是通过 AI Director Agent 理解用户输入的小说、文章、短句、脚本和参考素材，形成导演判断，组织角色卡、场景卡、风格卡、导演板、首尾帧、分镜草稿和视频生成工具，最终产出可复用、可迭代、可质检的视频项目资产和成片。

一句话定义：

> 让 AI 像导演一样思考，而不是像工具一样填空。

## 2. 产品目标

### 2.1 核心目标

从用户输入的一篇小说、一篇文章、一段短话或一组参考素材出发，经过 AI Director Agent 的导演判断和素材组织，最终产出一个视频项目，并逐步生成可用视频片段或完整视频。

### 2.2 产品差异

本产品不以“直接文生视频”为核心卖点，而以“导演驱动 + 全能参考 + 资产复用 + Agent 工作流”为核心。

与普通视频生成工具的区别：

| 普通视频大模型 | AI Director Agent Studio |
| --- | --- |
| 输入 prompt，输出视频 | 输入文本/素材，生成导演判断、资产、片段和成片 |
| 主要依赖单次生成 | 依赖项目资产、参考素材和多轮迭代 |
| 用户自己写提示词 | AI Director 自主判断如何拍 |
| 结果难复用 | 角色卡、场景卡、导演板、提示词、片段可复用 |
| 难解释为什么这样拍 | 输出导演理由和质检结果 |

## 3. 目标用户

### 3.1 第一阶段目标用户

1. 短剧、漫剧、微电影、剧情短视频创作者。
2. 想把小说、文章、故事片段改成视频的人。
3. 不擅长影视导演和分镜，但希望 AI 能帮忙完成视频表达的人。
4. 有素材积累，希望复用角色、场景、风格和参考图的人。

### 3.2 用户痛点

1. 不知道如何把文字内容转成有情绪、有镜头感的视频。
2. 不懂导演术语、分镜、景别、运镜、构图。
3. 直接文生视频随机性太强，不可控。
4. 过度写分镜和 prompt 又容易限制 AI 发挥。
5. 角色、场景、风格、首尾帧等素材缺少统一管理。
6. 生成结果不可复用，下一次又要重新描述。
7. 不知道视频好不好、问题在哪里、如何修改。

## 4. 核心用户旅程

```text
用户输入文本/素材
-> Agent Factory 判断需求并创建/选择 Video Production Agent
-> AI Director 生成导演判断
-> 系统生成或引用角色卡、场景卡、风格卡、导演板
-> 拆分 15 秒导演段落
-> 每段判断 Control Level 和生成方式
-> 调用图片/视频生成工具
-> 保存生成资产
-> AI Director 质检结果
-> 用户反馈
-> 项目继续迭代
```

## 5. 核心概念

### 5.1 VideoProject

VideoProject 是视频生产的项目容器，承载从原始输入到最终成片的所有资产和决策。

字段建议：

```text
VideoProject
├── id
├── title
├── brief
├── sourceMaterials
├── targetFormat
│   ├── 类型：短剧 / 漫剧 / 微电影 / 广告 / 剧情短视频
│   ├── 时长
│   ├── 画幅比例
│   └── 平台
├── directorDNA
├── directorNotes
├── characterBible
├── worldBible
├── styleBible
├── directorBoards
├── segments
├── referenceAssets
├── generatedAssets
├── evaluations
└── status
```

### 5.2 AI Director Agent

AI Director Agent 是核心主导者，负责导演判断，而不是机械拆分镜。

职责：

1. 理解文本内容和情绪。
2. 判断人物关系和戏剧冲突。
3. 选择是否需要分镜草稿。
4. 选择导演板和视听表达方式。
5. 判断每段的控制强度。
6. 组织参考素材。
7. 生成镜头意图和生成提示词。
8. 调用工具生成图片/视频。
9. 输出导演理由。
10. 质检成片是否服务情绪和叙事。

### 5.3 DirectorDNA

DirectorDNA 是 AI Director 的导演配置，不是大量硬规则，而是导演原则、知识、偏好和判断协议。

包括：

1. 导演类型。
2. 风格倾向。
3. 情绪表达偏好。
4. 镜头语言偏好。
5. 控制强度策略。
6. 是否偏向自由生成。
7. 是否偏向强一致性。
8. 质检标准。

### 5.4 Director Board

Director Board 是可复用导演板，用于表达某类戏如何拍。

示例：

1. 二人对话戏导演板。
2. 跑步追逐戏导演板。
3. 角色初登场导演板。
4. 情绪崩溃导演板。
5. 暧昧拉扯导演板。
6. 空景转场导演板。

### 5.5 Director Notes

Director Notes 是当前项目或当前段落的导演手记。

它回答：

1. 这段戏的核心情绪是什么？
2. 观众应该感受到什么？
3. 为什么这样拍？
4. 哪些地方需要强控制？
5. 哪些地方应该让 AI 自由发挥？

### 5.6 15s Director Segment

15 秒导演段落是视频生产的基本工作单元。

字段建议：

```text
DirectorSegment
├── id
├── order
├── duration
├── plotPurpose
├── emotionCurve
├── characterRelationChange
├── audienceFocus
├── directorNotes
├── selectedDirectorBoards
├── controlLevel
├── generationMode
├── requiredReferences
├── shotIntentions
├── promptPack
├── generatedAssets
└── evaluation
```

### 5.7 Reference Asset

Reference Asset 是可复用的视频生产资产。

类型：

1. 原文素材。
2. 角色卡。
3. 角色三视图。
4. NPC 卡。
5. 场景卡。
6. 色彩卡。
7. 风格参考。
8. 首帧/尾帧。
9. 分镜草稿。
10. 导演手记。
11. 导演板。
12. 镜头提示词。
13. 生成视频片段。
14. 质检记录。

### 5.8 Control Level

控制强度用于平衡“可控性”和“AI 创造力”。

| 等级 | 含义 | 适用场景 |
| --- | --- | --- |
| loose | 只给情绪、关系、风格，让 AI 自由发挥 | 梦境、氛围、意识流 |
| guided | 给导演手记、导演板、角色/场景卡 | 对话、情绪戏、一般剧情 |
| strict | 给首尾帧、参考图、明确动作 | 角色初登场、连续动作 |
| locked | 给分镜草稿、固定构图、固定动作 | 产品展示、强一致性镜头 |

## 6. 功能需求

### 6.1 Agent Factory

目标：帮助用户创建或选择合适的 Video Production Agent / AI Director Agent。

需求：

1. 用户可以描述想做的视频类型、输入材料、目标平台、风格偏好。
2. Factory 主动澄清关键信息。
3. Factory 推荐 Video Production Agent 配置。
4. Factory 推荐 DirectorDNA。
5. Factory 推荐默认 Director Boards。
6. Factory 推荐需要的模型、skills、tools。
7. Factory 生成初始 VideoProject 结构。
8. Factory 创建前应给出测试/发布判断。

### 6.2 Video Project 工作台

目标：围绕一个视频项目组织生产过程。

需求：

1. 创建项目。
2. 上传或输入源文本。
3. 绑定参考素材。
4. 查看项目 Brief。
5. 查看 AI Director 的导演判断。
6. 查看 15s 段落列表。
7. 查看角色卡、场景卡、风格卡。
8. 查看生成资产。
9. 查看质检结果。
10. 支持项目继续迭代。

### 6.3 AI Director 对话区

目标：让用户与 AI Director 围绕项目进行沟通。

需求：

1. 用户可以输入修改意见。
2. 用户可以 `@素材` 引用角色、场景、首尾帧、视频片段。
3. AI Director 先输出导演判断，再输出执行建议。
4. AI Director 必须说明导演理由。
5. AI Director 可以建议需要补充哪些参考素材。
6. AI Director 可以主动判断是否需要分镜草稿。
7. AI Director 可以建议使用文生视频、图生视频、首尾帧或多参考图。

### 6.4 Director Notes

目标：记录 AI Director 对项目和段落的导演理解。

需求：

1. 支持生成项目级导演手记。
2. 支持生成段落级导演手记。
3. 支持用户编辑或确认导演手记。
4. 支持将导演手记保存为资产。
5. 支持后续生成时引用导演手记。

### 6.5 Director Board Library

目标：管理可复用导演板。

需求：

1. 内置基础导演板。
2. 支持用户创建自定义导演板。
3. 支持 AI Director 推荐导演板。
4. 支持按场景类型筛选导演板。
5. 支持导演板进入项目引用。
6. 支持导演板版本化。

MVP 内置导演板建议：

1. 二人对话戏导演板。
2. 角色初登场导演板。
3. 情绪崩溃导演板。
4. 跑步追逐戏导演板。
5. 空景转场导演板。

### 6.6 Reference Asset Library

目标：统一管理全能参考素材。

需求：

1. 上传图片、视频、文本、文件。
2. 保存 AI 生成结果为资产。
3. 为资产设置类型：角色卡、场景卡、风格卡、首帧、尾帧、分镜草稿等。
4. 资产可被项目引用。
5. 资产可被 Agent 对话 `@素材` 引用。
6. 资产保留来源血缘：项目、Agent、会话、消息、生成任务。
7. 删除项目或 Agent 不应物理删除已保存资产。

### 6.7 Character / Scene / Style Cards

目标：把参考素材结构化。

需求：

1. 从文本中提取角色列表。
2. 生成人物角色卡。
3. 支持角色图、三视图、不同场景参考。
4. 生成场景卡。
5. 生成色彩/风格卡。
6. 支持手动编辑。
7. 支持在视频生成中作为参考输入。

### 6.8 Segment Planner

目标：将视频项目拆成 15 秒导演段落。

需求：

1. AI Director 根据剧情和目标时长拆分段落。
2. 每段包含剧情目的、情绪曲线、人物关系变化、观众注意力中心。
3. 每段选择 Control Level。
4. 每段选择生成方式。
5. 每段绑定参考素材。
6. 每段生成提示词包。
7. 每段生成后进入质检。

### 6.9 Generation Tools

目标：接入可执行生成能力。

需求：

1. 文生视频。
2. 图生视频。
3. 首尾帧视频生成。
4. 多图参考视频生成。
5. 图片生成。
6. 角色图生成。
7. 场景图生成。
8. 视频任务轮询。
9. 生成结果自动保存为资产。
10. 不同 Agent/项目可配置不同模型列表。

### 6.10 Director Evaluation

目标：判断生成结果是否满足导演意图。

需求：

1. 检查情绪是否成立。
2. 检查镜头是否服务叙事。
3. 检查角色是否一致。
4. 检查参考素材是否被正确使用。
5. 检查是否过度模板化。
6. 检查是否过度控制，导致 AI 创造力下降。
7. 给出修改建议。
8. 支持将失败结果转为反例资产。

## 7. 页面需求

### 7.0 界面风格与交互原则

从当前项目界面可以反推出，产品不应该做成营销型首页，也不应该做成简单聊天机器人页面，而应该是一个长期使用的专业生产工作台。

整体界面气质应是：

```text
专业、克制、清晰、资产密集、流程可见、对话驱动、右侧资产常驻。
```

不要做成：

1. 大面积宣传 hero 页面。
2. 一问一答式聊天页。
3. 纯表单型后台。
4. 复杂影视软件式重操作界面。
5. 上来就是时间线和剪辑器。

应该做成：

```text
左侧：项目 / Agent / 会话 / 段落导航
中间：AI Director 对话与当前生产任务
右侧：Reference Asset Library / 当前段落资产 / 生成结果
```

当前项目已有的界面意图应该延续：

1. 左侧 sidebar 承载 Agent、会话和项目入口。
2. 中间区域是主要工作流，不是静态详情页。
3. 右侧素材面板常驻，支持按 Agent / 项目 / 类型筛选。
4. 输入框支持 `@素材`，说明素材引用是核心交互。
5. 模型切换在输入区附近，说明生成模型是当前任务上下文的一部分。
6. Agent Factory 是“左侧对话 + 右侧步骤”的创建工作台。
7. Agent 设置页是 tab 化配置，不是单页长表单。
8. 布局宽度和面板状态应记忆，适合长期生产使用。

### 7.0.1 整体视觉风格

建议视觉方向：

1. 背景以白色、浅灰为主，避免强烈装饰感。
2. 使用细线、轻分割、浅色标签表达层级。
3. 卡片只用于资产、段落、候选 Agent、生成任务，不要让整个页面都变成卡片堆。
4. 重点信息密度要高，适合创作者反复查看和比较。
5. 避免大面积渐变、装饰图形、营销插画。
6. 按钮和状态要明确：生成中、已保存、需确认、可发布、需修改、失败。
7. 视觉上要更像“创作控制台 / 导演工作台”，而不是“聊天应用”。

### 7.0.2 首页形态

首页不应该是产品介绍页，而应该直接进入工作状态。

推荐首页：

```text
左侧：项目与 Agent 导航
中间：当前 VideoProject / AI Director 工作区
右侧：Reference Asset Library
```

当没有项目时，空状态也不应是营销文案，而应该引导用户：

1. 输入一段小说 / 文章 / 短句。
2. 上传已有素材。
3. 让 Agent Factory 创建 AI Director 项目。

### 7.0.3 AI Director 工作区

中间主区域不是普通聊天流，而是“对话 + 结构化产物”的混合工作区。

AI Director 的回复应该有结构化块：

```text
导演判断
导演理由
建议补充的参考素材
15s 段落建议
生成方式建议
下一步动作
```

不要让所有内容都淹没在 Markdown 长文本里。重要产物应该可以一键保存为资产：

1. 保存为导演手记。
2. 保存为角色卡。
3. 保存为场景卡。
4. 保存为导演板。
5. 保存为段落方案。
6. 保存为提示词包。

### 7.0.4 右侧资产面板

右侧资产面板是这个产品的核心，不是附属附件栏。

它应该常驻显示：

1. 当前项目资产。
2. 当前段落资产。
3. 最近生成结果。
4. 用户上传参考。
5. 角色卡 / 场景卡 / 风格卡。
6. 首帧 / 尾帧 / 视频片段。

资产卡片要显示：

```text
资产类型
名称
来源
状态
被引用次数
所属项目 / 段落
是否已保存
```

右侧资产面板应该支持：

1. 列表 / 网格切换。
2. 按类型筛选。
3. 按项目筛选。
4. 按段落筛选。
5. 拖入或插入到当前输入。
6. 作为参考素材绑定到当前段落。

### 7.0.5 15s Segment 视图

Segment 不应该一开始做成复杂时间线，而应该做成可扫描的段落列表。

推荐形态：

```text
段落 01 · 0-15s
情绪：压迫 -> 反击
Control Level：guided
生成方式：图生视频 + 角色参考
状态：待生成 / 已生成 / 需修改 / 已通过
```

点击段落进入详情，显示：

1. 剧情目的。
2. 情绪曲线。
3. 导演手记。
4. 导演板。
5. 参考素材。
6. 提示词包。
7. 生成结果。
8. 质检意见。

### 7.0.6 Agent Factory 界面

Agent Factory 应该保持当前项目已有的思路：

```text
左侧：需求澄清对话
右侧：创建过程
  1. 需求澄清
  2. 推荐 AI Director / Video Agent
  3. 配置 DirectorDNA / AgentDNA
  4. 测试与创建
```

Factory 生成的候选不只是 Agent 名称，还应展示：

1. 推荐的导演类型。
2. 适合的视频类型。
3. 默认 Director Boards。
4. 默认 Skills / Tools。
5. 推荐模型。
6. 适合的 Control Level 策略。

### 7.0.7 配置页风格

配置页应延续当前 Agent 设置页的 tab 结构，但面向 AI Director 增加领域配置。

推荐 tabs：

```text
基础信息
DirectorDNA
导演板
模型与生成方式
Skills / Tools
资产与记忆
测试与质检
发布与删除
```

配置页不要做成全局大表单。每个 tab 只处理一类决策，并且要说明该配置影响哪里。

### 7.0.8 过程可见，但不暴露隐藏推理

界面应该展示 AI Director 的工作过程，但不展示隐藏推理链。

应该展示：

1. 正在分析文本。
2. 正在判断情绪。
3. 正在选择导演板。
4. 正在查找参考素材。
5. 正在生成角色卡。
6. 正在提交视频任务。
7. 正在保存资产。
8. 正在质检结果。

不应该展示：

1. 模型内部长推理。
2. 无法审计的思维链。
3. 对用户无意义的 token 级日志。

这个过程面板应该更接近“导演生产日志”，而不是技术 debug log。

### 7.0.9 参考界面的取舍建议

从参考截图和当前项目界面可以提炼出一些值得借鉴的结构，但不能直接照抄。

#### 值得保留的结构

1. 三栏任务工作台。

```text
左侧：项目 / Agent / 会话 / 功能导航
中间：主任务流 / AI Director 对话 / 结构化产物
右侧：任务监控 / 素材资产 / 项目状态
```

这与当前项目已有的 sidebar、chat-area、artifact-panel 高度一致，也符合 AI Director 产品的长期创作场景。

2. 底部输入常驻。

输入框应该像任务控制台，而不是普通聊天框。它需要承载：

```text
自然语言指令
@素材 引用
模型切换
生成模式切换
附件上传
快捷命令
当前项目 / 段落上下文
```

3. Activity / Steps 折叠展示。

工具调用、素材读取、视频任务提交、质检过程不应全部铺开。默认展示摘要：

```text
Activity: 3 steps
读取素材 / 生成角色卡 / 提交视频任务
```

用户需要时再展开看细节。

4. 右侧监控面板。

参考图中的“任务监控 / 产物 / 技能与 MCP”对本项目很有启发，但应改造成：

```text
导演监控
├── 当前段落进度
├── 待确认事项
├── 生成任务
├── 最新资产
├── 绑定导演板
├── 使用模型
└── 质检风险
```

5. 模型选择器。

参考图中的模型下拉非常适合当前项目。但本项目不能只是文本模型列表，而应按生成场景过滤：

```text
文本分析模型
图像生成模型
视频生成模型
多模态理解模型
质检评估模型
```

并且会话中只能切换当前 Agent / 当前项目允许的模型。

#### 需要避免的误区

1. 不要复制“通用 AI 助手”的空首页。

类似“不止聊天，搞定一切”的中心化空状态适合通用工具，但本产品应该更具体：

```text
创建视频项目
输入故事文本
上传角色 / 场景 / 风格参考
让 AI Director 写导演判断
```

2. 不要让右侧变成普通文件列表。

当前截图中的右侧文件监控对编程助手有用，但本项目右侧必须是导演资产语义：

```text
角色卡
场景卡
风格卡
首帧
尾帧
段落视频
导演手记
质检记录
```

3. 不要把步骤日志做得太技术化。

“Browser / Exec / Process”适合通用 Agent，但 AI Director 应展示用户能理解的生产动作：

```text
分析故事
判断情绪
选择导演板
查找参考素材
生成角色卡
生成视频片段
质检镜头
```

4. 不要过度居中留白。

通用 AI 首页可以留大量空白，但视频生产工作台应该资产密集、状态清晰。空白只适合未创建项目时的引导页。

5. 不要一开始做重型剪辑器。

参考图都是任务和资产工作台，不是专业剪辑软件。本产品早期也应避免上来做复杂 timeline，而是先做好 Segment 列表、资产面板和生成结果。

#### 建议的最终界面骨架

```text
┌──────────────────────────────────────────────────────────────┐
│ Top Bar: 当前项目 / 搜索 / 设置 / 模型状态                     │
├──────────────┬──────────────────────────────┬────────────────┤
│ 左侧导航      │ 中间 AI Director 工作流        │ 右侧导演监控     │
│              │                              │                │
│ Projects     │ Director Chat                │ Segment Status │
│ Agents       │ Director Notes               │ Assets         │
│ Segments     │ Segment Cards                 │ Tasks          │
│ Assets       │ Generation Results            │ Evaluation     │
│ Boards       │                              │ Boards         │
│ Settings     │ Bottom Command Input          │                │
└──────────────┴──────────────────────────────┴────────────────┘
```

这个结构能同时承载：

1. ChatGPT / Codex 式的自然语言任务流。
2. Zcode / Solo-Web 式的任务监控和产物面板。
3. 当前项目已有的 Agent、会话、素材、模型绑定能力。
4. AI Director 产品独有的导演手记、导演板、15s 段落和参考资产。

## 8. MVP 范围

### 8.1 MVP 必做

1. VideoProject 基础模型。
2. Reference Asset 类型扩展。
3. AI Director Agent 的基础 DNA。
4. Director Notes 生成。
5. 15s Segment 生成。
6. Control Level 判断。
7. 基础 Director Board Library。
8. 角色卡、场景卡、风格卡生成。
9. 文生视频 / 图生视频 / 首尾帧至少接入一种或两种。
10. 生成结果保存为资产。
11. 基础 Director Evaluation。

### 8.2 MVP 可暂缓

1. 完整多 Agent 协作。
2. 复杂 timeline 剪辑。
3. 自动合成长视频。
4. 高级音频、配音、音乐。
5. 完整 MCP 生态。
6. 复杂导演风格市场。
7. 多人协作。

## 9. 非目标

第一阶段不做：

1. 不做泛用 Agent 平台。
2. 不做单纯视频模型壳。
3. 不做复杂多 Agent 团队编排。
4. 不做专业剪辑软件替代品。
5. 不要求一次生成完整长片。
6. 不强制用户理解导演术语。
7. 不把分镜作为默认最高层控制方式。

## 10. 验收标准

### 10.1 产品验收

1. 用户输入一段小说或文章后，系统能生成导演判断。
2. 系统能说明为什么这样拍，而不是只给 prompt。
3. 系统能生成角色卡、场景卡或风格卡中的至少两类资产。
4. 系统能把内容拆成 15s 段落。
5. 每个段落有 Control Level 和生成方式建议。
6. 用户可以引用参考素材参与生成。
7. 生成结果能保存为资产并在后续引用。
8. 系统能对生成结果给出导演维度的质检意见。

### 10.2 体验验收

1. 用户不需要懂分镜，也能得到可理解的导演方案。
2. 用户可以看到 AI Director 的判断理由。
3. 用户可以选择接受、修改或重新生成导演判断。
4. 用户不会被迫填写复杂影视表单。
5. 用户能清楚知道当前项目有哪些资产、哪些段落、哪些生成结果。

## 11. 后续演进

### Phase 1：AI Director MVP

目标：跑通导演判断到生成片段的闭环。

重点：

1. VideoProject。
2. Director Notes。
3. Director Board。
4. Reference Asset。
5. 15s Segment。
6. 基础生成工具。

### Phase 2：资产复用与质检增强

目标：让项目资产真正可复用。

重点：

1. 角色一致性。
2. 场景一致性。
3. 风格一致性。
4. Director Evaluation。
5. 失败案例反哺。

### Phase 3：专业子 Agent

目标：把复杂环节升级为独立 Agent。

可能的子 Agent：

1. Story Analysis Agent。
2. Character Bible Agent。
3. Visual Style Agent。
4. Storyboard Draft Agent。
5. Video QA Agent。

### Phase 4：完整视频生产流

目标：从片段生成升级到完整视频项目。

重点：

1. Timeline。
2. 剪辑合成。
3. 配音字幕。
4. 音乐音效。
5. 完整成片导出。

## 12. 关键原则

1. AI Director 主导导演判断。
2. 人类提供目标、素材、反馈和边界。
3. 系统提供知识、资产、工具、记忆和质检。
4. 不机械套用导演模板。
5. 不默认强制分镜。
6. 先输出导演理由，再执行生成。
7. 所有生成结果都应可沉淀为资产。
8. Agent、素材、模型、项目、段落都要有清晰边界。

## 13. 最终建议

产品第一阶段不应该追求“大而全的视频生成平台”，而应该聚焦：

```text
输入文本/素材
-> AI Director 做导演判断
-> 生成项目资产
-> 拆 15s 段落
-> 调用生成工具
-> 保存结果
-> 质检和迭代
```

这条路径既保留 AI 大模型的创造力，又能通过资产、导演板、DirectorDNA 和质检机制建立产品壁垒。
