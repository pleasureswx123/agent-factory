# AI Director Agent 产品指导思想

日期：2026-06-23  
定位：对“AgentOS / Agent Factory / 视频生产 Agent”方向的产品意图总结  
目标：把当前讨论中形成的核心判断沉淀为后续产品、架构和实现的指导文档

## 1. 核心结论

这个项目真正要做的不是普通的“文生视频”，也不是简单的“多 Agent 协作平台”，而是：

> 一个由 AI Director Agent 主导的、参考素材驱动的、可复用资产沉淀的视频生产工作台。

更准确地说，系统目标不是：

```text
输入一句话 -> 调用视频大模型 -> 输出一个视频
```

而是：

```text
小说 / 文章 / 短句 / 用户素材
-> AI Director 理解情绪、人物关系、叙事目的和视觉动机
-> 自主判断该如何拍
-> 组织角色卡、场景卡、风格卡、参考图、首尾帧、历史素材
-> 调用图像 / 视频 / 多模态模型和工具
-> 生成可复用、可迭代、可质检的视频项目资产
-> 最终产出视频
```

所以项目的护城河不是“能生成视频”，而是：

> 能像导演一样理解内容、组织素材、判断镜头语言、控制情绪表达，并让 AI 视频模型在不过度束缚的情况下发挥创造力。

## 2. 为什么不能只叫“文本到视频”

“文本到视频”容易把产品带偏。市面上很多视频大模型已经可以文生视频，如果产品只是输入文本然后生成视频，就会变成模型能力的薄壳。

当前项目真正关心的是“全能参考”和“生产过程”：

1. 用户可能输入小说、文章、短句、脚本。
2. 用户可能已有角色图、场景图、风格参考、历史素材。
3. 视频生成可能需要文生视频、图生视频、首尾帧、多图参考、角色参考、场景参考、风格参考。
4. 生成结果需要进入素材库，后续可复用。
5. 视频不是一次生成完成，而是经过分析、设定、导演判断、镜头组织、生成、质检、修改。

因此更准确的定位是：

```text
Reference-driven Video Agent Studio
Director-driven Video Agent Studio
参考驱动的视频 Agent 工作台
导演驱动的视频 Agent 工作台
```

## 3. 最重要的产品哲学

不要把 AI 变成一个执行人类死规则的工具。

系统不应该这样设计：

```text
两人对话必须正反打
强势方必须低机位
15 秒必须拆 5 个镜头
每段必须远景、中景、特写
每个视频都必须先画九宫格分镜
```

这会把 AI 限制成“执行表格的人”，而不是导演。

正确方向是：

```text
给 AI 足够好的导演知识、素材上下文、目标、边界和评估机制，
让 AI 自己判断要不要正反打、要不要分镜、要不要强控制、要不要自由发挥。
```

核心原则：

1. 给知识，不给死命令。
2. 给判断协议，不给固定模板。
3. 给导演知识库，不给僵硬规则库。
4. 让 AI 解释导演理由，而不是机械输出镜头表。
5. 分镜是工具，不是最高层控制对象。
6. 情绪、叙事、人物关系、视觉动机才是最高层控制对象。

## 4. AI Director Agent 是主导

系统中应该存在一个核心 Agent：

```text
AI Director Agent / 导演 Agent
```

它不是普通的“分镜 Agent”，也不是单纯的“提示词 Agent”。它的职责是做导演判断。

AI Director Agent 应该回答：

1. 这段内容的核心情绪是什么？
2. 这段内容的戏剧冲突是什么？
3. 谁是观众注意力中心？
4. 当前人物关系是压迫、疏离、暧昧、反转，还是失控？
5. 这段应该克制表达，还是强烈表达？
6. 需要强控制画面，还是应该让 AI 视频模型自由发挥？
7. 是否需要角色卡、场景卡、风格参考、首尾帧或多图参考？
8. 是否需要分镜草稿？如果不需要，为什么？
9. 适合用哪些导演手法？为什么？
10. 哪些常见手法不适合？为什么？
11. 最终视频如何判断是否成功？

这不是限制 AI，而是让 AI 进入“导演思考状态”。

## 5. 导演手记、导演板、分镜草稿的区别

### 5.1 导演手记

导演手记是当前项目或当前段落的导演判断。

它回答：

```text
这段戏为什么这样拍？
观众应该先感受到什么，再感受到什么？
哪个角色的情绪是核心？
镜头应该克制还是激进？
哪里需要留白？
哪里需要爆点？
哪里必须控制，哪里让 AI 发挥？
```

示例：

```text
这场戏的重点不是两个人说了什么，而是女主第一次意识到自己被操控。
前半段不要频繁切镜头，让男主的语速和空间压迫感占据画面。
女主沉默时使用浅景深近景，不需要台词。
最后一句反击不要给大动作，用移焦从男主转到女主，让权力关系完成反转。
```

### 5.2 导演板

导演板是可复用的导演方法模板，是“这类戏通常怎么拍”的知识模块。

它不是具体分镜，而是导演知识、镜头策略、情绪策略和 AI 提示词范式的组合。

示例：

```text
二人对话戏导演板
跑步追逐戏导演板
角色初登场导演板
情绪崩溃导演板
暧昧拉扯导演板
反派压迫导演板
空景转场导演板
```

导演板相当于导演知识库，也可以理解为 AI 导演的“教官符”。

### 5.3 分镜草稿

分镜草稿只是辅助工具，不应该默认成为最高控制方式。

它适合在这些情况下使用：

1. 需要表达空间关系。
2. 需要保持角色动作连续性。
3. 需要首尾帧或图生视频。
4. 需要和用户对齐视觉想法。
5. 复杂动作或复杂场面需要提前规划。

但很多时候，过早拆分镜会让 AI 过度关注画面，反而丢掉情绪。

所以产品原则是：

```text
不要用分镜主导 AI。
要用导演意图主导 AI。
```

## 6. AI 导演的视听 API 库

“视听 API 库”不是技术 API，而是一套可被 AI Director 调用的导演知识模块。

### 6.1 景别 API

```text
远景：建立空间、孤独、渺小、命运感
全景：展示人物与环境关系
中景：人物行动、对话和关系
近景：进入情绪
特写：情绪爆点、细节暗示、注意力锁定
空景：停顿、转场、情绪余韵、环境压迫
```

### 6.2 机位与视角 API

```text
低机位：压迫、强势、英雄化、权力感
高机位：弱势、被审视、无助、失控
平视：真实、平等、冷静
过肩：关系、对抗、代入感
外反打：疏离、旁观、关系距离
同向同框：同盟、共谋、暧昧、共同面对
```

### 6.3 运镜 API

```text
推近：情绪逼近、压力上升
拉远：抽离、失落、命运感
前跟：推进目标、制造压迫或速度
后跟：追随、逃离、悬念
横移：关系并置、空间揭示
环绕：困住、迷恋、失控
手持：真实、紧张、不稳定
慢门抽帧：疲惫、速度、记忆感、梦境感
```

### 6.4 构图与空间 API

```text
前景遮挡：隔阂、窥视、压迫
留白：孤独、等待、压抑
对称构图：秩序、仪式、控制
失衡构图：焦虑、危险、失控
框中框：困住、被观察、关系限制
纵深空间：权力距离、追逐、空间压迫
```

### 6.5 光影与色彩 API

```text
逆光：神秘、距离、轮廓、命运感
侧光：冲突、分裂、暧昧
低照度：压抑、危险、隐秘
高调光：轻盈、干净、梦幻
冷色：疏离、理性、孤独
暖色：亲密、回忆、欲望
强对比：冲突、压迫、戏剧性
低饱和：现实、疲惫、压抑
```

### 6.6 焦点 API

```text
移焦：注意力转移、关系变化、信息揭示
浅景深：孤立人物、强化情绪
深焦：空间关系、群像调度、现实感
失焦：迷离、记忆、恍惚、不确定
```

### 6.7 镜头衔接 API

```text
动作接动作：保持动作连续
视线接视线：保持注意力连续
情绪接情绪：用情绪而不是空间完成转场
色彩接色彩：用视觉母题衔接场景
声音先行：用声音带入下一场
空景转场：给情绪留余韵
相似构图转场：用画面形态解决场景跳跃
```

这些 API 不应该被写成死规则，而是作为 AI Director 可选择的表达资源。

## 7. DirectorDNA

DirectorDNA 是导演 Agent 的核心配置，类似 AgentDNA，但更偏影视创作。

```text
DirectorDNA
├── identity
│   ├── 导演类型：短剧 / 漫剧 / 微电影 / 广告 / 电影感
│   ├── 风格倾向：现实主义 / 悬疑 / 情绪流 / 爽感短剧
│   └── 审美边界
├── narrative
│   ├── 叙事节奏
│   ├── 情绪曲线
│   ├── 信息释放方式
│   └── 视觉动机
├── visualLanguage
│   ├── 景别
│   ├── 机位与视角
│   ├── 构图与空间
│   ├── 光影与色彩
│   ├── 运镜
│   ├── 焦点
│   ├── 剪辑节奏
│   └── 特效手法
├── directingPrinciples
│   ├── 情绪优先
│   ├── 镜头服务叙事
│   ├── 不机械套用模板
│   ├── 必要时才做分镜
│   ├── 保留 AI 模型创造空间
│   └── 对导演选择给出理由
├── aiGenerationPolicy
│   ├── 什么时候强控
│   ├── 什么时候弱控
│   ├── 什么时候不用分镜
│   ├── 什么时候需要首尾帧
│   ├── 什么时候需要参考图
│   └── 什么时候让模型自由发挥
└── evaluation
    ├── 情绪是否成立
    ├── 角色是否一致
    ├── 镜头是否服务叙事
    ├── 节奏是否顺
    ├── 是否过度模板化
    └── 是否保留生成模型创造性
```

DirectorDNA 中可以有少量硬规则，但硬规则应该只用于边界，不应该写成大量拍摄公式。

建议硬规则：

```text
1. 不要把导演技巧当成固定公式。
2. 不要机械生成镜头清单。
3. 不要在没有必要时强制分镜。
4. 不要为了炫技牺牲情绪和叙事。
5. 所有镜头建议必须说明服务的情绪或叙事目的。
```

## 8. Control Level：控制强度

AI 视频生成的核心矛盾是：

```text
既要控制画面和一致性，
又不能过度控制导致 AI 失去创造力。
```

因此每个段落都应该有一个 Control Level：

```text
loose
  只给情绪、关系、风格，让 AI 自由发挥。

guided
  给导演手记、导演板、角色卡、场景卡、镜头意图。

strict
  给首尾帧、参考图、明确角色和动作约束。

locked
  给分镜草稿、固定构图、固定动作，适合强一致性场景。
```

Director Agent 应该自己判断每个段落的控制强度。

示例：

```text
情绪氛围段：loose
对话冲突段：guided
角色初登场：strict
动作连续性：strict
梦境 / 意识流：loose
产品展示：locked
```

## 9. 15 秒导演段落

AI 视频生成天然适合按短段落组织。建议用 15 秒作为基础生产单元。

每个 15s Director Segment 包含：

```text
15s Director Segment
├── 剧情目的
├── 情绪曲线
├── 角色关系变化
├── 观众注意力中心
├── 推荐导演板
├── 关键镜头意图
├── Control Level
├── 参考素材需求
├── 生成方式
│   ├── 文生视频
│   ├── 图生视频
│   ├── 首尾帧
│   ├── 角色参考
│   └── 多图参考
├── 禁止事项
└── 质检标准
```

重点不是机械拆镜头，而是：

```text
这 15 秒要让观众感受到什么？
AI Director 为什么选择这种拍法？
哪些地方必须一致？
哪些地方可以让 AI 发挥？
```

## 10. Reference Asset Library

当前项目已有 Artifact Panel 和 `@素材` 机制，这应该升级成导演资产库。

```text
Director Asset Library
├── 原文素材
├── 角色卡
├── 角色三视图
├── NPC 卡
├── 场景卡
├── 色彩卡
├── 风格参考
├── 首帧 / 尾帧
├── 分镜草稿
├── 导演手记
├── 导演板
├── 镜头提示词
├── 生成视频片段
└── 质检记录
```

这正好对应“全能参考”的方向。系统不是只靠文本生成，而是围绕资产持续组织和复用。

## 11. VideoProject

不要只围绕 Agent 和 Conversation 组织数据。真正的视频生产需要一个项目容器。

```text
VideoProject
├── projectBrief
├── sourceMaterials
├── referenceAssets
├── directorNotes
├── directorDNA
├── storyAnalysis
├── characterBible
├── worldBible
├── styleBible
├── directorBoards
├── segments
├── shotIntentions
├── promptPack
├── generatedAssets
├── timeline
├── finalVideo
└── evaluation
```

Agent 是生产者，Artifact 是资产，VideoProject 是容器。

如果没有 VideoProject，素材会散、会话会散、Agent 输出会散，最后很难形成真正的视频生产系统。

## 12. Agent 如何分层

不要一开始就做一堆子 Agent。正确路线是先跑通导演驱动的视频生产闭环，再逐步 Agent 化。

第一阶段：

```text
Agent Factory
  -> 创建 / 调优 Video Production Agent
  -> 选择 DirectorDNA
  -> 推荐导演板库、角色卡模板、场景卡模板、生成工具

Video Production Agent
  -> 主控整个视频项目
  -> 内置 Director Skill / Director Module
  -> 调用导演板、素材库、图像工具、视频工具、质检工具
```

第二阶段：

```text
Video Production Agent
  -> Director Agent
  -> Story Analysis Agent
  -> Character Bible Agent
  -> Visual Style Agent
  -> Storyboard Draft Agent
  -> Asset Generation Agent
  -> Video QA Agent
```

升级为子 Agent 的判断标准：

1. 这个环节需要独立长期记忆。
2. 这个环节有独立规则和评估标准。
3. 这个环节会在多个项目中复用。
4. 这个环节质量足够重要，需要单独调试和版本化。
5. 这个环节的输出会被多个下游环节依赖。

## 13. Skills 与 Tools 的边界

Skills 是导演能力、方法论和工作方式。

示例：

```text
小说分析 Skill
角色设定 Skill
情绪判断 Skill
导演判断 Skill
镜头语言 Skill
对话戏导演 Skill
动作戏导演 Skill
短视频节奏 Skill
质检 Skill
```

Tools 是可执行动作。

示例：

```text
读取素材
保存素材
生成角色卡
生成场景卡
生成首帧
生成尾帧
图生视频
文生视频
首尾帧视频生成
视频任务轮询
导出分镜文档
合成视频
```

不要把工具伪装成 Agent。图片生成、视频生成、上传、下载、轮询、转码，本质上都是工具。

## 14. Agent Factory 的新职责

Agent Factory 不应该只是创建一个普通 Agent。

它应该帮助用户创建：

1. Video Production Agent。
2. DirectorDNA。
3. Director Board Library。
4. Reference Asset Schema。
5. VideoProject 结构。
6. 生成流程。
7. 模型和工具绑定。
8. 质检标准。

Factory 要问的问题不是：

```text
你想创建什么 Agent？
```

而是：

```text
你想生产什么类型的视频？
输入材料是什么？
希望观众感受到什么？
有没有角色、场景、风格参考？
希望 AI 自由发挥多一些，还是一致性强一些？
产出是漫剧、短剧、微电影、广告片，还是剧情短视频？
```

## 15. 最小可行路线

建议下一阶段按这个顺序做：

```text
1. 引入 VideoProject 概念
2. 把 Artifact 升级为 Reference Asset / Director Asset
3. 定义 DirectorDNA
4. 定义 Director Board
5. 定义 Director Notes
6. 定义 15s Director Segment
7. 做一个 Video Production Agent
8. 让它先输出导演判断，而不是直接拆分镜
9. 接入图片 / 视频生成工具
10. 保存所有中间产物为可复用资产
11. 增加 Director Evaluation
12. 后续再拆独立子 Agent
```

第一阶段的目标不是一次生成完整大片，而是跑通：

```text
输入文本 / 素材
-> AI Director 生成导演判断
-> 生成角色卡 / 场景卡 / 风格卡
-> 拆成 15s 段落
-> 为每段选择控制强度和生成方式
-> 调用生成工具
-> 保存资产
-> 质检和迭代
```

## 16. 最终产品定义

推荐的产品定义：

> AI Director Agent Studio 是一个导演驱动的视频生产工作台。它通过 Agent Factory 创建具备导演判断能力的 Video Production Agent，让 AI 不只是生成画面，而是基于情绪、人物关系、视觉动机和参考素材，自主选择导演手法、组织镜头语言、调用生成模型，并沉淀可复用的视频生产资产。

一句话版本：

> 让 AI 像导演一样思考，而不是像工具一样填空。

## 17. 最关键的提醒

不要把系统做成：

```text
人类制定大量拍摄规则 -> AI 按规则执行 -> 输出机械视频
```

要做成：

```text
人类提供目标、素材和反馈
AI Director 进行导演判断
系统提供知识、资产、工具、记忆和质检
最终共同产出视频
```

你的角色不是替 AI 当导演，而是：

```text
目标设定者
素材提供者
审美反馈者
最终决策者
```

AI Director 的角色是：

```text
理解故事
判断情绪
选择视听表达
组织参考素材
调用生成工具
解释导演理由
根据反馈进化
```

这是当前项目最值得坚持的方向。
